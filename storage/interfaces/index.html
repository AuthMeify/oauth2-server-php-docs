Interfaces

Overview

This library supports adapters for several different storage engines. Among them is PDO (for MySQL, SQLite, PostgreSQL, etc), MongoDB, Redis, and Cassandra.

This is done through multiple PHP Interfaces which dictate how different objects are stored. Interfaces allow the library to be extended and customized for multiple platforms.

Access Token Interface

For implementing basic Access Tokens. This is required for the Token Controller, unless Crypto Tokens are used instead.

Authorization Token Interface

For implementing the Authorization Code Grant Type

Client Interface

Required for the Token Controller and Authorize Controller, and is used to gather information about the client making the request.

Client Credentials Interface

For validating a client's credentials. This is required for all requests to the Token Controller, in order to validate the client making the request.

Crypto Token Interface

An interface for using Crypto Tokens. This requires the Public Key interface (below), and is used for validating access tokens without the use of a database system.

JWT Bearer Interface

Required for the JWT Bearer Grant Type.

Public Key Interface

For implementing Crypto Token.

Refresh Token Interface

For implementing the Refresh Token Grant Type.

Scope Interface

For implementing scopes using the Scope Util.

User Credentials Interface

For implementing the User Credentials Grant Type.
